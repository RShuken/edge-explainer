Use Cases

How Dan Uses Edge

Specific examples of how Edge transforms Dan's daily operations, grounded in what we've discovered works best for Crusoe Ventures' workflow.

7am

"Edge, what's my day look like?"

Every morning at 7 AM, Dan gets a Telegram message with his full day context: who he's meeting, what they've discussed before (pulled from Notion People DB), which deals are active with those companies, overdue action items, and urgent unread emails.

How it works ↓

Queries People directory (20K+ records), Companies DB, and Task board. Unknown attendees get Tavily web search for LinkedIn and recent news.

Pre

"Prep me for my next meeting"

Before every external meeting, Edge researches attendees by cross-referencing calendar invites with Notion's People and Companies databases. For each person: their title, company, last interaction, recent company news, and any open deals or action items.

How it works ↓

Fuzzy matching (Jaro-Winkler) resolves "Dan C." to "Dan Caruso" across 20K+ records. Exact email match first, then probabilistic above 85% confidence.

15m

Email triage every 15 minutes

Edge checks edge@crusoeventures.com via Himalaya (permanent IMAP, no OAuth expiry). Classifies by urgency (Immediate/Today/This Week/FYI), category (Deal Flow/Portfolio/LP/Operations), and intent (Requires Response/Meeting Request/FYI).

How it works ↓

Dan's Telegram reactions ("Was Urgent"/"Not Urgent") train the classifier over time. Drafts never auto-send — always approve first.

Act

"Process my meeting notes"

After a Google Meet call, Gemini generates the transcript. Edge processes it: extracts decisions, commitments, and follow-ups. Matches mentioned people to Notion. Creates draft action items in the Task database marked "Pending Approval."

How it works ↓

Telegram inline buttons per action item (Approve/Reject). Only approved items become active Notion tasks. All decisions logged for SEC audit compliance.

CRM

Automatic CRM enrichment

After every meeting and email interaction, Edge enriches contact records in Notion. New title mentioned? Updated. Company change? Flagged. Last contact date? Auto-updated. The People and Companies databases stay current without Dan lifting a finger.

How it works ↓

4-database schema (People, Companies, Deals, Interactions) with relations and rollups. External IDs prevent duplicates during sync.

PR

5-pass AI code review on every PR

When code is pushed, GitHub Actions runs 5 parallel Claude-powered review passes: Security (OWASP Top 10), Performance, Code Quality, Architecture, and Test Coverage. All third-party actions pinned to commit SHAs to prevent supply chain attacks.

How it works ↓

All actions pinned to commit SHAs (not tags) after tj-actions/Trivy supply chain attacks. 5 parallel review passes cover security, performance, quality, architecture, and tests.

What the Experts Say

Research-Backed Design Decisions

Every architectural choice in Edge is informed by leading AI researchers and security experts.

Simon Willison

"Typing code is no longer the bottleneck; human comprehension is. Set pace limits on how much agents generate, aligned with your ability to actually review it."

Applied: Dan operates "on the loop" — he refines rules and policies, not individual outputs.

Andrej Karpathy

"You don't optimize the output, you optimize the instructions. The human's role shifts from writer to director to research advisor."

Applied: Dan's preferences and brand voice are stored in immutable config files that Edge reads but cannot modify.

Anthropic Engineering

"Claude is already smart enough — intelligence is not the bottleneck, context is. Use just-in-time loading rather than pre-loading everything."

Applied: Each skill is a standalone script with focused context. No persistent sessions that accumulate and overflow.

Cisco Security Research

"Personal AI agents are a security nightmare. 135,000+ OpenClaw instances were found exposed on the public internet. 512 vulnerabilities identified, 8 critical."

Applied: Edge is bound to localhost, uses FileVault encryption, scoped credentials per service, and nightly security audits.

SEC FY2026 Priorities

"AI usage policies and supervision are a top examination priority. Firms must document how AI is used in investment processes and retain all AI outputs."

Applied: Every Edge action is logged with timestamp, input, output, and approval status. 7-year retention for compliance.

Martin Fowler / ThoughtWorks

"When unsatisfied with results, 'in the loop' practitioners fix artifacts directly. 'On the loop' practitioners change the harness that produced the artifact."

Applied: Edge's behavior is governed by config files Dan can tune, not by reviewing every email draft individually.

Trust & Safety

Security Architecture

Six layers of defense protect Dan's data and ensure Edge operates within strict, auditable boundaries.

1

FileVault Encryption

Full-disk encryption on the Mac Mini hardware. Data at rest is protected even if the physical device is compromised.

2

Scoped Credentials

Each service integration uses its own API key with minimal required permissions. No single key grants broad access.

3

Approval Gates

Human-in-the-loop for all write operations. Dan must approve actions via Telegram before Edge modifies any external system.

4

Content Sanitization

All external inputs are sanitized against prompt injection attacks. Edge validates and cleans data before processing.

5

Security Council

Nightly automated audit by four AI personas: Auditor, Red Team, Compliance, and Privacy. Issues are flagged for human review.

6

Audit Logging

Every action Edge takes is recorded with timestamps, inputs, outputs, and reasoning. Full traceability for any operation.

Security Council v2

Layer 0 — Foundation • Nightly at 3:30 AM • 1,866 lines

Approved

The Business Problem

Dan's AI assistant has access to email, calendar, Notion, and 20,000+ contacts. A single misconfiguration — an exposed port, a leaked API key, an overly permissive file — could trigger SEC enforcement, erode LP trust, or expose material non-public information across portfolio companies. Cisco found 135,000+ exposed OpenClaw instances with 512 known vulnerabilities. Manual security reviews don't scale when the system changes daily.

Why This Approach

Single-perspective reviews miss blind spots. Four specialized AI personas analyze the system independently, then a realism filter removes noise:

Offensive Security

Thinks like an attacker. Finds exploitable ports, weak auth, prompt injection vectors, credential leaks, privilege escalation.

Defensive Security

Checks firewall, FileVault, file permissions, updates, backups, network segmentation, logging, TLS.

Data Privacy

Hunts for PII in logs, overly broad scoping, stale credentials, unencrypted databases, secrets in git history.

Operational Realism

The false-positive filter. Adjusts severity for context, assesses blast radius, dismisses noise. A Mac Mini behind an office router ≠ a public cloud server.

How It Works

3:30 AM

Baseline collection — 11 system checks (ports, permissions, processes, npm audit, FileVault, firewall)

4 Passes

Sequential persona analysis via Claude Sonnet API, then realism filter

7-Day Dedup

SHA-256 hash deduplication — only new findings are reported

Instant Alert

Critical findings trigger immediate Telegram alert, not batched

Implementation

Standalone Node.js Claude Sonnet API SQLite (3 tables) launchd scheduling Telegram alerts Dedicated alert group ~$0.20/night

Layer 0.1 — Security Hardening

Layer 0 — Foundation • Deployed March 31 • 7 steps

Deployed

The Business Problem

Edge runs on a Mac Mini with access to email, calendar, Notion, and 20K+ contacts. Before any credentials touch the machine, the disk must be encrypted, the gateway locked to localhost, commands restricted to an allowlist, and defenses deployed against prompt injection and credential leaks.

Why This Approach

Security assessment required FileVault before any credential deployment. Cisco research found 135K+ exposed instances. Content sanitizer blocks 13 prompt injection patterns from emails and transcripts. Secret redaction auto-strips API keys from all output. Pre-commit hook prevents .env and credential files from entering git.

How It Works

FileVault

Full-disk encryption enabled. Data at rest protected even if hardware is stolen.

Gateway Lockdown

Bound to 127.0.0.1 only. Tailscale for remote access, never exposed to network.

Exec Allowlist

23 approved commands. Everything else requires human approval via Telegram.

Content Defense

13 sanitizer patterns + 14 secret redaction patterns + pre-commit git hook.

Implementation

FileVault Gateway 127.0.0.1 exec-approvals 13 sanitizer patterns 14 redaction patterns pre-commit hook Deployed

Philosophy

Key Principles

The design philosophy that governs every decision in the Edge system.

"Human on the loop"

Dan refines rules and policies, not individual outputs. Edge operates autonomously within defined boundaries, escalating only when needed.

"Security first"

Every integration uses scoped access. Every write operation requires approval. Every action is logged. Defense in depth, not perimeter-only.

"Standalone scripts"

Each skill is a self-contained script that runs to completion. No persistent context windows. No token overflow. Clean execution every time.

"Notion is the hub"

All roads lead to Notion. It is the source of truth for people, companies, tasks, and interactions. Every other system syncs to it.

"Documentation is a deliverable"

Every skill has an SOP. Every decision has a rationale. Every integration has a wiki page. The system documents itself as it operates.
